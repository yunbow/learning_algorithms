CityHash

アルゴリズムの概要

CityHashは、Googleによって開発された高速なハッシュ関数で、特に短い文字列や中程度の長さの文字列に対して優れたパフォーマンスを示します。このハッシュ関数は、高速な計算と良好な分散特性を両立させ、現代のCPUアーキテクチャに最適化されています。

基礎知識

CityHashは、2011年にGoogleのGeoff Pike氏とJyrki Alakuijala氏によって開発されました。このハッシュ関数は、主に64ビットCPUでの高速な実行を目的としており、MurmurHashの後継的な位置づけとなっています。CityHashは、32ビット、64ビット、128ビット、256ビットなど、複数のハッシュ長を出力するバリアントが存在します。

CityHashの基本的な設計思想は、現代のCPUアーキテクチャを最大限に活用することです。特に命令パイプラインや複数命令の並列実行、キャッシュラインの活用などを考慮して設計されています。また、バイトオーダー（エンディアン）に中立的なアルゴリズムとなっています。

用語説明

  • ハッシュ関数（hash function）: 任意のサイズのデータを固定長の値に変換する関数
  • 衝突（collision）: 異なる入力が同じハッシュ値を生成する状況
  • 分散性（distribution）: ハッシュ値が出力範囲全体にどれだけ均等に分布しているか
  • アバランシェ効果（avalanche effect）: 入力の小さな変化が出力に大きな変化をもたらす特性
  • シード値（seed value）: ハッシュ計算の初期値として使用される値
  • ワードサイズ（word size）: CPUが一度に処理できるデータの単位（通常32ビットまたは64ビット）
  • エンディアン（endianness）: マルチバイトデータの順序表現（ビッグエンディアン/リトルエンディアン）
  • バイト単位操作（byte-level operations）: バイト単位でのビット演算
  • ハッシュテーブル（hash table）: ハッシュ関数を使用してデータを格納し、高速アクセスを実現するデータ構造

特徴

CityHashの主な特徴は以下の通りです：

  • 64ビットCPUアーキテクチャに最適化された設計
  • 特に短い文字列と中程度の長さの文字列に対して優れたパフォーマンス
  • 高いアバランシェ効果を持ち、入力の小さな変化でも出力が大きく変化する
  • 優れた分散特性により、ハッシュテーブルでの衝突が少ない
  • 暗号学的に安全ではないが、一般的なハッシュテーブル用途では十分な強度
  • 複数のビット長（32、64、128、256）の出力をサポート
  • シード値を使用して異なるハッシュ結果を生成可能
  • エンディアンに依存しない実装
  • 命令パイプラインを効率的に利用するよう最適化されている
  • 多くの種類のCPUで高速に動作するよう設計されている

適用ケース

CityHashは以下のような場面で特に有用です：

  • 大規模なハッシュテーブルやハッシュマップの実装
  • データベースのインデックス作成と高速検索
  • 重複データの検出と除外
  • キャッシュシステムのキー管理
  • 分散システムでのデータの分配と負荷分散
  • 文書の類似性検出（MinHash等と組み合わせて）
  • コンテンツアドレス可能ストレージシステム
  • ネットワークパケットの高速処理
  • 大量の小さな文字列を効率的に処理するアプリケーション
  • データチェックサムの生成（データ整合性の確認）

アルゴリズムの手順

具体的な手順

CityHash64の基本的な手順は以下の通りです：

1. 初期化

  1. 入力文字列のサイズを確認する
  2. 文字列の長さに応じた定数（マジックナンバー）を初期値として設定する
  3. 必要に応じてシード値を組み込む

2. 文字列長に応じた処理の分岐

  1. 文字列長が0〜4バイトの場合：
    • 各バイトを特殊な方法で組み合わせて混合する
    • 短い文字列専用の単純なハッシュ計算を実行する
  2. 文字列長が5〜12バイトの場合：
    • 文字列を最大3つの32ビットブロックとして処理
    • 各ブロックに特殊なビット演算と混合関数を適用
  3. 文字列長が13〜24バイトの場合：
    • 文字列を最大3つの64ビットブロックとして処理
    • 各ブロックを混合して中間ハッシュ値を生成
  4. 文字列長が24バイト以上の場合：
    • 文字列を64ビットブロックごとに処理
    • 複数のハッシュ値を計算して並行に進める
    • 特殊な混合関数によりブロック間の依存関係を作り出す

3. メイン処理ループ（長い文字列の場合）

  1. 文字列を64ビットブロックに分割して処理
  2. 各ブロックごとに以下の処理を実行：
    • 64ビットブロックを読み込む
    • ブロックに対して複数の演算（シフト、乗算、XOR等）を適用
    • 既存のハッシュ値と組み合わせて新しいハッシュ値を生成

4. 残りのバイトの処理

  1. 64ビットアライメントに収まらなかった残りのバイトを処理
  2. 残りのバイト数に応じて適切なビット演算を適用

5. 最終的なミキシング（ファイナライズ）

  1. 中間ハッシュ値に対して複数の最終処理を適用：
    • ビットシフト操作を適用
    • 特定の定数との乗算を実行
    • XOR操作によるビットの混合
    • アバランシェ効果を高めるための追加の混合処理

6. 結果の返却

  1. 最終的なハッシュ値（通常は64ビット整数）を返す

計算量

CityHashアルゴリズムの計算量は以下の通りです：

時間計算量

  • 平均の場合: O(n) - 入力文字列の長さに比例
  • 最悪の場合: O(n) - 入力文字列の長さに比例

空間計算量

  • 全体: O(1) - 入力サイズに関係なく定数量のメモリを使用

CityHashの最大の強みは、その高いパフォーマンスです。特に短い文字列に対して非常に効率的に動作し、64ビットアーキテクチャ上で最適化されています。入力データを一度だけスキャンするため、時間計算量はO(n)となります。

また、CityHashは高度に最適化された実装により、一定の長さの入力に対して他の多くのハッシュ関数と比較して少ないCPUサイクルで処理を完了できます。特に、現代のCPUアーキテクチャ上でのパイプライン処理、キャッシュの利用効率、分岐予測に配慮した設計となっているため、理論的な計算量以上の実性能を発揮します。

CityHashは、暗号学的に安全なハッシュ関数（SHAなど）と比較すると、セキュリティ面では劣るものの、パフォーマンス面では大幅に優れています。このため、暗号学的な安全性が必要ない一般的なハッシュテーブルの実装や、大量のデータ処理が必要なアプリケーションに適しています。

なお、CityHashの後継としてGoogle社はFarmHashを開発しており、さらに高度な最適化と幅広いプラットフォームへの対応が図られています。また、高速なハッシュ関数としてはXXHashも広く使用されており、特定のケースではCityHashを上回るパフォーマンスを示すこともあります。
