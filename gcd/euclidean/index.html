ユークリッドの互除法

アルゴリズムの概要

ユークリッドの互除法は最大公約数（GCD）を求めるための古典的なアルゴリズムで、紀元前300年頃にユークリッドによって「原論」で記述されました。このアルゴリズムは単純さと効率性を兼ね備え、数千年経った現代でも広く使用されています。

基礎知識

最大公約数（Greatest Common Divisor: GCD）とは、2つ以上の整数が共通して持つ約数のうち最大のものを指します。例えば、24と36の最大公約数は12です。

ユークリッドの互除法の基礎となる数学的原理は、「2つの数のGCDは、大きい方の数から小さい方の数の整数倍を引いても変わらない」というものです。これを繰り返し適用することで、効率的にGCDを求めることができます。

用語説明

  • 除数（divisor）: ある数を割り切ることができる数
  • 商（quotient）: 割り算の結果得られる値
  • 余り（remainder）: 割り算を行った後に残る値
  • 互いに素（coprime/relatively prime）: 最大公約数が1である2つの整数の関係
  • 再帰（recursion）: アルゴリズムが自分自身を呼び出す技法

特徴

ユークリッドの互除法の主な特徴は以下の通りです：

  • 単純明快な数学的基盤に基づいている
  • 再帰的なアプローチが可能
  • 除算と剰余演算のみを使用
  • 対数時間で収束する効率的なアルゴリズム
  • 拡張ユークリッドの互除法として発展し、モジュラ逆数や不定方程式の解法にも応用可能
  • 数学教育において再帰と数論の概念を教えるための優れた例

適用ケース

ユークリッドの互除法は以下のような場面で特に有用です：

  • 分数の約分（既約分数への変換）
  • 暗号理論（特にRSA暗号などの公開鍵暗号システム）
  • モジュラ逆数の計算（拡張ユークリッドの互除法を使用）
  • リニアディオファントス方程式の解法
  • 連分数展開の計算
  • 有限体における演算
  • 一般的な数学的計算や問題解決

アルゴリズムの手順

具体的な手順

ユークリッドの互除法の基本的な手順は以下の通りです：

  1. 2つの非負整数a, b（ただしa ≥ b > 0とする）を入力として受け取る
  2. aをbで割った余りrを計算する: r = a mod b
  3. もしr = 0ならば、bが最大公約数であるため、bを返して終了する
  4. そうでなければ、a = b, b = rとして、ステップ2に戻る

この過程は必ず有限回のステップで終了し、最終的にbの値として最大公約数が得られます。

具体例: GCD(48, 18)を計算する場合

  1. a = 48, b = 18
  2. r = 48 mod 18 = 12
  3. r ≠ 0 なので、a = 18, b = 12 として繰り返す
  4. r = 18 mod 12 = 6
  5. r ≠ 0 なので、a = 12, b = 6 として繰り返す
  6. r = 12 mod 6 = 0
  7. r = 0 となったので、b = 6 が最大公約数

したがって、GCD(48, 18) = 6 となります。

計算量

ユークリッドの互除法の計算量は以下の特性を持ちます：

  • 時間計算量: O(log(min(a,b)))
  • 空間計算量:
    • 反復版: O(1) - 定数個の変数のみを使用
    • 再帰版: O(log(min(a,b))) - 再帰の深さに比例

ユークリッドの互除法が対数時間で収束する理由は、連続する2回の操作で少なくとも一方の数が半分以下になることが保証されているためです。これにより、アルゴリズムは非常に大きな数に対しても効率的に動作します。

特に、ラメの定理によれば、ユークリッドの互除法における除算の回数は、小さい方の数の十進表記の桁数の約5倍を超えないことが証明されています。

この効率性により、暗号理論や数論における大きな整数を扱う場合でも実用的な時間内で計算が可能となっています。
